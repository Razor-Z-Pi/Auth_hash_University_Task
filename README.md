# Auth_hash_University_Task
Популярные протоколы шифрования и хеширования / лабораторная 

# Задание 

При реализации backend части, всегда приходится думать о безопасности для разных уровней и этапов использования приложения. Рассмотрим один пример. При регистрации на сайте вы вводите логин, пароль и тд., разумеется когда пароль приходит на сервер он не хранится в открытом виде, он хешируется и результат сохраняется в БД. Когда же мы входим в аккаунт и вводим логин и пароль, то пароль хешируется и сумма хеша сравнивается с тем, что хранится в БД.
Ваша задача создать две формы при помощи  js, php, css.
Первая форма отвечает за регистрацию пользователя и содержит поля для ввода логина, пароля и имени пользователя. 
Вторая форма - логина, она содержит поля только для ввода логина и пароля.
После реализации форм, добавьте бизнес логику приложения. Для хранения паролей используйте mysql.
После того как вы создали БД, создайте в ней одну таблицу User с такими столбцами как username, password, login, тип данных определите исходя из логики применения (login уникальное поле). Данные поля будут использоваться в нашем  приложении.
После этого на стороне приложения реализуйте логику добавления пользователей в вашу локальную БД, это зависит от языка.
Далее когда пользователь вводит данные для регистрации и нажимает кнопку регистрация, соберите данные из полей и добавьте их в БД, но при этом не забудьте хешировать пароль. Вот пример того, что вы можете получить login = “Pavel@mail.ru”, password = “a665a45920422f9d417e4867efdc4f”, username = “Pavel”. Затем просто при помощи sql запроса или при помощи ORM(почитайте что это) сохраните данные в вашу локальную БД.
Когда пользователь вводит эти данные на форме логина, также соберите данные с полей, и при помощи sql запроса найдите пользователя по логину из вашей БД, и сравните хеш паролей из БД с введенным на форме логина, если он совпадает напишите на экране сообщение “Привет {username}” (поздоровайтесь с пользователем по имени).
